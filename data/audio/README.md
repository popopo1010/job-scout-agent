# 音声データディレクトリ

このディレクトリには、CAの営業通話音声データとその処理結果を保存します。

## ディレクトリ構造

```
data/audio/
├── pending/          # 未処理の音声ファイル（ここにアップロード）
├── processing/       # 処理中（文字起こし中）
├── transcripts/      # 文字起こし済みテキスト（自動生成）
├── processed/        # 処理完了（音声ファイル）
└── failed/           # 処理失敗
```

## 音声ファイルの命名規則

```
{YYYY-MM-DD}_{CA_ID}_{会議識別子}.{拡張子}

例:
2025-01-15_CA001_client-call-001.m4a
2025-01-15_CA002_weekly-mtg.mp3
```

## 対応形式

- `.m4a` (iOS録音、Zoom録音) - 推奨
- `.mp3` (一般的な音声形式)
- `.wav` (高品質音声)
- `.webm` (ブラウザ録音)

## 使い方

1. 音声ファイルを `pending/` に配置
2. 文字起こし処理が自動実行される（実装後）
3. 処理完了後、音声ファイルは `processed/` に移動
4. 文字起こしテキストは `transcripts/` に保存
5. 既存のフィードバックシステムが自動的にテキストを処理

## 注意事項

- 音声ファイルには個人情報が含まれる可能性があるため、適切に管理してください
- ファイルサイズは最大500MBまで
- 通話時間は最大2時間まで
