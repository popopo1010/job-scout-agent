## 📋 概要

現在の営業セグメント分析は「企業規模別（大/中/小）」「エリア別」の2軸のみで、解像度が粗い状況です。

本提案では、**事業形態（実際の工事内容・顧客種別）**を基軸としたセグメント分析を行い、データドリブンな営業戦略の立案を可能にします：

1. **事業形態別の分析** - 太陽光、RC、木造、民間工事、公共工事など実際の事業形態で分類
2. **各セグメントの平均給与・求人数分析** - インディードなどの求人データから実給与を分析
3. **成長率・伸び予測** - 各事業形態セグメントの将来性評価
4. **営業優先順位の明確化** - どこの事業形態セグメントを狙うべきかの判断基準

---

## 🎯 提案する分析フレームワーク

### 事業形態ベースのセグメント分類

電気工事業界の実際の事業形態に基づいて、以下の観点からセグメントを整理します：

#### 主要分析軸

1. **事業形態（工事種別・顧客種別）** - メイン軸
2. **エリア（都道府県 × 都市規模）** - サブ軸
3. **年収層（求人数ベースの分布）** - サブ軸
4. **企業規模** - 補助軸

## 📊 事業形態セグメントの仮分類（データ収集前に仮設定）

以下の事業形態セグメントを想定しますが、**実際のデータ収集・分析により適切に調整**します：

### 1. 建築構造別セグメント

| セグメント | 定義 | 特徴 |
|-----------|------|------|
| **RC（鉄筋コンクリート）工事** | RC造建築物の電気工事 | ビル、マンション等、安定した需要 |
| **木造工事** | 木造住宅・建築物の電気工事 | 戸建て住宅、在来工法等 |
| **鉄骨工事** | S造（鉄骨造）建築物の電気工事 | 工場、倉庫、大型施設等 |
| **プレハブ工事** | プレハブ建築物の電気工事 | 仮設、モバイル施設等 |

### 2. 工事種別セグメント

| セグメント | 定義 | 特徴 |
|-----------|------|------|
| **新築工事** | 新規建築物の電気工事 | 設計から施工まで、規模が大きい |
| **リフォーム工事** | 既存建築物の改修・増設工事 | 小規模・多様な要求 |
| **メンテナンス・保守** | 既存設備の点検・保守・修理 | 定期・継続的な取引 |
| **設備更新** | 既存設備の取り替え工事 | 老朽化対策、省エネ化 |

### 3. 顧客種別セグメント

| セグメント | 定義 | 特徴 |
|-----------|------|------|
| **民間工事（一般）** | 民間企業・個人向け工事 | オフィス、店舗、住宅等 |
| **公共工事** | 官公庁・公共施設向け工事 | 学校、病院、官公庁、道路等 |
| **設備管理** | ビル・施設の設備管理業務 | 定期点検、保守、運用 |

### 4. 専門分野別セグメント

| セグメント | 定義 | 特徴 |
|-----------|------|------|
| **再エネ・EV設備** | 太陽光パネル設置・保守工事、EV充電スタンド設置工事 | 成長領域、脱炭素政策で需要増 |
| **データセンター** | データセンター内の電気工事 | 高単価、専門性が高い |
| **電気通信工事** | 通信設備・LAN配線工事（データセンター以外） | 5G・IoT需要で成長 |
| **防災・セキュリティ** | 防災設備・セキュリティ設備工事 | 安定した需要 |
| **空調・給排水関連** | 空調設備・給排水設備の電気工事 | 建築設備の一環 |

### 5. 事業規模別セグメント

| セグメント | 定義 | 特徴 |
|-----------|------|------|
| **大規模工事** | 大型建築物・施設の工事 | 高単価、長期プロジェクト |
| **中規模工事** | 中規模建築物・施設の工事 | 標準単価、中期プロジェクト |
| **小規模工事** | 小規模建築物・改修工事 | 低単価、短期プロジェクト |

## 🔍 データ収集・分析計画

### ステップ1: 各種求人サイトからのデータ収集

以下の求人サイトから、電気工事士の求人データを収集します：

1. **インディード（Indeed）**
   - 求人数: 最大1,000-2,000件を目標
   - 収集項目: 求人名、企業名、年収、勤務地、事業形態（求人内容から抽出）

2. **リクナビNEXT**
   - 既存のスクレイピングデータを活用
   - 追加でデータ収集

3. **工事士.com**
   - 電気設備業界専門サイト
   - 事業形態の記載が詳細な可能性

4. **その他の求人サイト**
   - マイナビ転職、doda、en等
   - 必要に応じて追加収集

### ステップ2: 事業形態の自動分類

収集した求人データから、以下を実施：

1. **キーワード抽出**
   - 求人名、求人内容から事業形態を示すキーワードを抽出
   - 例：「太陽光」「RC」「木造」「住宅」「ビル」「データセンター」等

2. **事業形態の自動分類**
   - キーワードに基づいて自動分類
   - 複数のセグメントに該当する場合は複数分類

3. **手動レビュー・補正**
   - 自動分類の精度を確認
   - 分類ルールの調整
   - 必要に応じて手動分類

### ステップ3: セグメント別分析

各事業形態セグメントごとに以下を分析：

1. **求人数**
   - 各セグメントの求人数
   - 全体に占める割合

2. **平均年収**
   - 各セグメントの平均年収・中央値
   - 最大値・最小値
   - 年収層別分布

3. **年収層別求人数分布図**
   - 各セグメントの年収層別求人数
   - ヒストグラムで可視化

4. **成長率分析**
   - 過去のデータがあれば成長率を算出
   - 政府統計・業界レポートと照合

### 3. 事業形態別成長率分析

**実施内容**:
- 過去の求人数推移を分析（可能な範囲で）
- 各事業形態の成長率（CAGR）を算出

**出力例**:
```

- 再エネ・EV設備: +8-12%/年（脱炭素政策による需要増、太陽光+EV充電）

- 木造工事: +1-2%/年（微増）
- データセンター: +5-8%/年（デジタル化需要）
- 電気通信工事: +5-8%/年（5G・IoT需要、データセンター以外）
...
```

### 4. 営業優先順位マトリックス

**実施内容**:
- 各事業形態セグメントの「求人数」「平均年収」「成長率」を評価
- 3軸のマトリックスを作成
- 優先順位（S/A/B/C）を決定
- 営業リソース配分の指針を作成

**評価軸**:
- **求人数**: 多いほど市場規模が大きい
- **平均年収**: 高いほど単価ポテンシャルが高い
- **成長率**: 高いほど将来性がある

**出力例**:
```

  → 求人数: 中、平均年収: 高、成長率: 高
- RC工事 × 全国 × 標準年収層
  → 求人数: 多、平均年収: 中、成長率: 中

B級（選択的）:
- 木造工事 × 全国 × 標準年収層
  → 求人数: 中、平均年収: 中、成長率: 低
```

## 💡 想定される課題と対策

### 課題1: 事業形態の分類精度

**対策**:
- キーワードベースの自動分類を実装
- 分類が難しい場合は手動分類も検討
- 複数の事業形態に該当する場合は複数分類
- 分類ルールを段階的に改善

### 課題2: 求人サイトのスクレイピング制約

**対策**:
- 既存のスクレイピング機能を活用
- 利用規約を遵守
- 必要に応じて手動収集も検討
- 複数のサイトからデータを収集して補完

### 課題3: 年収データの不完全性

**対策**:
- 年収幅がある場合は中央値を採用
- 日給・月給の場合は年収に換算
- データが不十分なセグメントは注記
- 複数のデータソースから補完

### 課題4: 事業形態の網羅性

**対策**:
- 実際のデータから新しい事業形態を発見
- 分類が見落としがないか確認
- 必要に応じてセグメントを追加・統合
