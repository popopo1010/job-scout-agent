# 使用定義書（仕様定義書）

## 1. Mission（ミッション）

### 1.1 プロジェクトの使命

**人の最適なマッチングをすることで、産業の生産性と働きがいを向上させ建設業界をハッピーにする。ひいては日本の生産性をあげていく。**

### 1.2 ビジョン（Vision）

#### FY25: 電気工事士の事業のPMF

**定性目標**: 電気工事士の事業のPMF達成

**定量目標**:
- 月内定8名
- 単月売上1,000万円
- 単価175万円 / 売上1,400万円/月
- CA 3名
- Sales 1名

#### FY26: 電気工事士業界No1、施工管理人材紹介のPMF

**定性目標**: 
- 電気工事士業界No1
- 施工管理人材紹介のPMF達成

**定量目標**:
- リード300名
- 月内定27名
- 単価175万円 / 売上4,725万円（年間5億6,700万円）
- CA 6名
- Sales 1名

#### FY27: 電気工事人材紹介の完全移譲、施工管理人材紹介の業界No1ポテンシャル

**定性目標**:
- 電気工事人材紹介の完全移譲
- 施工管理人材紹介の業界No1ポテンシャル確立

**定量目標**:
- リード400名
- 月内定36名
- 単価175万円 / 売上6,300万円（年間7億5,600万円）
- 事業責任者1名
- CA 9名（TL 2名、Player 7名）
- Sales 1名

### 1.3 価値提供

1. **データドリブンな意思決定**: セグメント掛け合わせ分析による期待展開率の正確な算出
2. **CAスキル向上**: AIによる営業通話分析と構造化されたフィードバック
3. **機会損失の防止**: 新規法人・新規エリアの自動検出による開拓機会の最大化

## 2. プロジェクト概要

### 2.1 プロジェクト名

**人材紹介事業 AIエージェントシステム（Job Scout Agent）**

### 2.2 対象ドメイン

- **業種**: 人材紹介業（電気工事士の転職支援）
- **主要資格**: 電気工事士（第一種・第二種）
- **対象地域**: 全国47都道府県

### 2.3 システム構成

本システムは3つの独立したサブシステムで構成される：

```
┌─────────────────────────────────────────────────────────────┐
│              人材紹介事業 AIエージェントシステム              │
├───────────────┬───────────────┬─────────────────────────────┤
│  システム1     │  システム2     │  システム3                  │
│  経営分析基盤  │  フィードバック │  求人データ整備             │
│               │  システム      │                            │
├───────────────┼───────────────┼─────────────────────────────┤
│ ・期待展開率   │ ・通話分析     │ ・新規法人発見              │
│ ・適正保有件数 │ ・FB生成      │ ・新規エリア検出            │
│ ・リード振分け │ ・Slack通知   │ ・自社求人比較              │
└───────────────┴───────────────┴─────────────────────────────┘
```

## 3. システム1: 経営分析基盤

### 3.1 目的

リードの期待展開率をセグメント掛け合わせで算出し、CA別の適正保有件数とリード振り分けを最適化する。

### 3.2 セグメント定義（4分類）

| セグメントID | 名称 | 条件 | 期待展開率 | 優先度 |
|-------------|------|------|-----------|--------|
| A | 資格あり若手 | 電気工事士資格あり & 40歳以下 | 75% | 1 |
| B | 資格ありベテラン | 電気工事士資格あり & 40歳超 | 60% | 2 |
| C | 資格なし若手 | 電気工事士資格なし & 40歳以下 | 40% | 3 |
| D | 資格なしシニア | 電気工事士資格なし & 40歳超 | 20% | 4 |

### 3.3 主要機能

- セグメント自動分類（資格有無×年齢）
- 適正保有件数算出
- リード振り分け支援
- パフォーマンス可視化

---

## 4. システム2: フィードバックシステム

### 4.1 目的

営業通話の書き起こしを分析し、CAのスキル向上を支援する構造化されたフィードバックを自動生成する。

### 4.2 主要機能

- 書き起こしファイルの自動読み込み
- PSS/ADSスキルに基づく通話分析
- AIによるフィードバック生成
- Slackへの自動通知

### 4.3 使用資料

- CA向け営業マニュアル
- PSS（Professional Selling Skills）資料
- ADS（Adaptive Dealer Selling）スキル資料

---

## 5. システム3: 求人データ整備

### 5.1 目的

電気工事士求人を網羅的に収集し、新規法人の発見と既存法人の新規エリア展開を自動検出する。

### 5.2 主要機能

- 求人サイトからの自動スクレイピング
- 新規法人の検出
- 既存法人の新規エリア検出
- 自社保有求人との比較
- 定期チェックと通知

### 5.3 比較ロジック

```
収集求人
  ↓
法人マッチング
  ├─→ 新規法人 → 新規法人リスト
  └─→ 既存法人
        ↓
      エリア比較
        ├─→ 新規エリア → 新規エリアリスト
        └─→ 既存エリア
              ↓
            求人比較 → 不足求人リスト
```

## 8. 経営P（経営計画）

### 8.1 Marketing（マーケティング）

**あるべき状態（定性）**: 期待展開率の高い求職者を集めることができている状態

**あるべき状態（定量）**:
- リードからの展開率が8%になっている
- 集客300名

**戦略**:
- SNS広告、リスティング広告を月1,000万円以上使える人材
- ショート動画の脚本を書ける人材

### 8.2 Cost（コスト）

**あるべき状態（定性）**: 許容マーケコストを維持できる単価を取れる顧客を明確にできている状態

**あるべき状態（定量）**:
- 電気工事士：平均単価175万円
- 施工管理：平均単価240万円

**戦略**:
- 求職者にとって価値ある企業を選定する
- XCHANGEにとっても価値ある企業を選定する
- 料率を高く取れるためのロジックを構築する

### 8.3 CA（キャリアアドバイザー）

**あるべき状態（定性）**: CAの期待展開率をクリアし、生産性が担保されている状態

**あるべき状態（定量）**:
- リードからの展開率8%
- 保有アクティブリード：23名/月
- アクティブ化率50%

**戦略**:
- 決められたOPSを徹底できる人材
- ナイスコミュニケーション
- 求職者との信頼関係をつくれるヒアリング
- 感情に寄り添えるコミュニケーション
- アクションをマニュアルに落とし込める
- 書類選考、面接の合否データを貯めることができる

### 8.4 RA（リクルーティングアドバイザー）

**あるべき状態（定性）**: ターゲットエリアにおいて求職者に適切な案件が担保されている状態

**あるべき状態（定量）**:
- 東京、神奈川、埼玉、千葉、大阪、名古屋、福岡にて注力案件が5つはある状態
- 電気工事士（未経験）で施工管理になれる案件が5つ
- 電気工事士（施工管理経験あり）で年収500万円以上獲得できる求人がエリアごとに5つ以上

**戦略**:
- 顧客の課題をヒアリングできる
- ニーズを理解し、該当求職者を紹介できる
- 営業のマニュアルをアップデートし続ける
- CAニーズを捉えて、柔軟に獲得をする

### 8.5 OPS（オペレーション）

**あるべき状態（定性）**: 定量理解するためのDBが設計、メンバーが確認して改善ができる状態を構築する

**あるべき状態（定量）**:
- ターゲットごとに、リードから内定受諾までのプロセスを見える化できる
- 上記をCAごとに見れるようになっている
- 求職者セグメント別に各会社の展開率が見えるようになっている
- dailyで着地を終えるようになっており、アクションに反映ができる
